# GameFlowManager

## Purpose

`GameFlowManager` owns the current Room 304 gameplay state flow.

## Responsibilities

- Start in `Investigation`.
- Enter `Deduction` after all required clues are collected.
- Enter `Combat` after correct deduction.
- Pass the active `BossDefinition` to boss spawning.
- Spawn and track the boss.
- Stop combat when the boss dies.
- Show reward selection from the active reward pool.
- Show chapter completion from active case data.
- Expose temporary debug actions.

No public fields. Serialized references are generated by `CombatSandboxCreator`.

## Usage Example

```csharp

```

## Common Pitfalls

- New scenes should use `GameFlowManager`, not legacy `Room304GameStateController`.
- Boss death flow depends on the spawned boss having `Damageable`.
- Missing case data falls back to legacy serialized fields only for compatibility.
- `ChapterComplete` currently does not load a new scene.
